## 原生组件

本笔记按照组件类型与 `type` 分类罗列原生 HTML 表单元素的常见属性，便于在 React 中结合 `useState`、`useReducer` 等 Hook 查阅。

### 1. 表单容器组件
**`<form>`**
- `action` / `method`：提交地址与方法。
- `enctype` / `target`：文件上传或新窗口提交。
- `noValidate`：关闭浏览器内置校验。
- `autoComplete`：`on` / `off` 控制整表自动填充。
- 事件：`onSubmit`、`onReset`。

**`<fieldset>` / `<legend>`**
- `fieldset disabled`：禁用字段组内全部控件。
- `legend`：字段组标题，增强结构与无障碍语义。

### 2. `<input>` 通用属性
- `name`：字段唯一标识，配合 `FormData`。
- `value` / `defaultValue`：受控与非受控值。
- `placeholder`、`autoComplete`、`autoFocus`、`spellCheck`：交互体验。
- `required`、`readOnly`、`disabled`：状态控制。
- `min`、`max`、`minLength`、`maxLength`、`pattern`、`step`、`title`：校验约束。
- `form*` 系列：`form`、`formAction`、`formMethod`、`formEnctype`、`formTarget`、`formNoValidate`。
- `list`：关联 `<datalist>` 自动补全。
- 事件：`onChange`、`onInput`、`onBlur`、`onFocus`、`onInvalid`。

### 3. `<input>` 按 `type` 分类
**文本输入类**
- `text`：默认文本输入。
- `password`：隐藏字符；属性同 `text`。
- `search`：附带清除按钮。
- `email`：格式校验、移动端弹出邮件键盘。
- `url`：校验 URL。
- `tel`：无格式校验但提示电话键盘。
- 常用附加属性：`inputMode`（`numeric`、`email` 等）、`enterKeyHint`。

**数字与范围类**
- `number`：上下箭头调节；支持 `min`、`max`、`step`；JS 可读取 `valueAsNumber`。
- `range`：滑块；同样使用 `min`、`max`、`step`；常绑定 `onInput` 实时预览。
- `color`：颜色拾取器，返回十六进制值。

**日期时间类**
- `date`、`time`、`datetime-local`、`month`、`week`：对应不同日期/时间选择器。
- 公共属性：`min`、`max`、`step`（控制时间步长，如分钟间隔）。

**选择/开关类**
- `checkbox`：多选或开关；使用 `checked`/`defaultChecked` 控制；JS 可设置 `element.indeterminate = true`。
- `radio`：单选；通过 `name` 分组，`value` 为提交值。
- `switch`：无原生类型，通常通过 `checkbox` 样式化。

**按钮提交类**
- `submit`：提交表单。
- `reset`：重置表单。
- `button`：自定义行为，React 中建议显式声明 `type="button"`。
- 可组合 `form*` 属性指定目标表单。

**文件/媒体类**
- `file`：文件上传。
  - `accept`：限制文件类型（如 `image/*`、`.pdf`）。
  - `multiple`：多文件上传。
  - `capture`：移动端指定摄像头。
  - JS 属性：`files`（`FileList`）。
- `image`：图片形态的提交按钮，需要 `src`、`alt`、`width`、`height`。

**隐藏字段类**
- `hidden`：在界面不可见，常用于提交附加数据。

### 4. 文本区域 `<textarea>`
- `rows`、`cols`：默认显示行列。
- `wrap`：`soft`（仅视觉换行）或 `hard`（换行记入值）。
- `maxLength`、`placeholder`、`autoComplete`、`spellCheck`。
- `value` / `defaultValue` + `onChange` 控制内容。

### 5. 选项选择组件
**`<select>`**
- `value` / `defaultValue`：受控或非受控。
- `multiple`：是否多选；受控时传数组。
- `size`：可见选项数量。
- `disabled`、`required`：状态控制。

**`<option>`**
- `value`：提交值；缺省时使用文本内容。
- `label`：自定义展示文本。
- `disabled`、`selected`：禁用或指定默认选项。

**`<optgroup>`**
- `label`：组标题。
- `disabled`：禁用整组。

**`<datalist>`**
- 与 `input[list]` 配合提供候选列表。

### 6. 辅助与显示组件
**`<label>`**
- `htmlFor`：绑定目标输入的 `id`。
- 将输入放在标签内可隐式关联，点击即聚焦。

**`<output>`**
- `for`：引用参与计算的控件 `id`。
- `value`：展示计算结果。

**`<progress>`**
- `value`、`max`：任务进度。
- `form` 属性可关联表单。

**`<meter>`**
- `value`、`min`、`max`、`low`、`high`、`optimum`：表示测量值范围与理想区间。

示例（可查看 `src/pages/home/home.tsx` 中 “辅助与显示组件示例”）：

```tsx
const [valueA, setValueA] = useState(10);
const [valueB, setValueB] = useState(20);
const [progress, setProgress] = useState(40);
const [load, setLoad] = useState(60);

<form onSubmit={(e) => e.preventDefault()}>
  <label htmlFor="value-a">数值 A</label>
  <input
    id="value-a"
    type="number"
    value={valueA}
    onChange={(e) => setValueA(Number.parseFloat(e.target.value) || 0)}
  />

  <label htmlFor="value-b">数值 B</label>
  <input
    id="value-b"
    type="number"
    value={valueB}
    onChange={(e) => setValueB(Number.parseFloat(e.target.value) || 0)}
  />

  <output htmlFor="value-a value-b">{valueA + valueB}</output>
</form>

<progress value={progress} max={100} />
<meter value={load} min={0} max={100} low={40} high={80} optimum={60} />
```

### 7. 校验、可访问性与事件
- 校验属性：`required`、`pattern`、`min`、`max`、`step`、`minLength`、`maxLength`、`title`、`inputMode`。
- 可访问性：`aria-*`、`role`、`aria-live`、`aria-describedby`、`aria-invalid`。
- 焦点与交互：`tabIndex`、`onFocus`、`onBlur`、`onChange`、`onInput`、`onInvalid`。

### 8. React 中的实践要点
- 受控优先：使用 `value`/`checked` + `onChange` 与 state 同步。
- 批量字段：结合 `name` 与对象/Reducer 管理，避免重复 `useState`。
- 利用原生属性减少自定义校验代码。
- 注意在 React 事件中使用 `event.target` 读取最新值，或使用函数式更新。

掌握这些原生特性后，可以在 React 中更高效地构建表单，并根据需求决定使用受控/非受控模式或结合第三方库。
